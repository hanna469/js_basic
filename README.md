## 연결 연산자(+) 더하기연산자(+)
* 문자열 + 문자열 > 문자열 결합 > 연결 연산자로 인식
* 문자열 + 숫자 > 문자열 결합 > 연결 연산자로 인식
* 숫자 + 숫자 > 숫자 결합 > 더하기 연산자로 인식
* (숫자+숫자)+문자열 > ()는 숫자 결합 뒤는 문자열 결합으로 인식
* -,*,/ 연산자 > 문자열 결합되어도 -,*,/ 연산자는 숫자형으로 자동형변환하여 인식함
## 산술연산자 :: 피연산자가 숫자인 경우
* `a + b` : 더하기
* `a - b` : 빼기
* `a * b` : 곱하기
* `a / b` : 나누기) 왼쪽 피연산자를 오른쪽 피연산자로 나눈 몫
* `a % b` : 나머지) 왼쪽 피연산자를 오른쪽 피연산자로 나눈 나머지
* `a ** b` : 거듭제곱)왼쪽 피연산자의 오른쪽 피연산자의 값만큼 곱한 값
* 계산할 수 없는 경우는 NaN 처리 ex) 'two' * 2 = NaN
## 산술 단항 연산자 :: 증감연산자
* `++a` : a를 1 증가시킨 후 a 연산 진행
* `a++` : a 연산 진행 후 a를 1 증가시킴
* `--a` : a를 1 감소시킨 후 a 연산 진행
* `a--` : a 연산 진행 후 a를 1 감소시킴
* 공통주의사항 : 연산자 뒤 데이터 종류는 반드시 숫자여야 함
    * b = ++a : b에 1만큼 증가시킨 a를 대입
    * b = a++ : b에 a를 대입한 뒤 결과에 1을 증가시킴
## 복합대입연산자
* 코드의 간결함을 위해 사용 | 사칙연산 모두 적용 가능
* 연산자:    예시   >   예시 뜻    예시 풀이
* `+=` : `a += b` > `a = a + b` 기존 a값에 b를 더하여 a에 대입
* `-=` : `a -= b` > `a = a - b` 기존 a값에 b를 빼서 a에 대입
* `*=` : `a *= b` > `a = a * b` 기존 a값에 b를 곱해서 a에 대입
* `/=` : `a /= b` > `a = a / b` 기존 a값에 b를 나눠서 a에 대입
* `%=` : `a %= b` > `a = a % b` 기존 a값에 b를 나눈 나머지값을 a에 대입

        * `const 변수명 = document.querySelectorAll('2개 이상 존재하는 태그명');`
        * `const 변수명 = document.querySelectorAll('2개 이상 존재하는 클래스명');`
        * 아이디에는 사용하지 않고 태그 또는 클래스가 같은 부모 요소 안 2개 이상 존재 시 해당 요소들을 필요에 따라 모두 선택해야할 때 All 키워드를 함께 사용한다.
        * `변수명[index];` //get~ 선택방식과 동일하게 인덱스로 2개 이상의 대상 구분 필요.
# DOM + 이벤트연결
1. `<태그 on이벤트="실행함수();"></태그>`
    * `<button onclick="alert();">`
    * 장점) 태그에 바로 이벤트를 작성할 수 있어서 간편함.
    * 단점) 태그에 내부스크립트로 작성된거라 반복 등 활용 불가능. 내부에서만 제한적 사용.
2. `DOM변수.addEventListener('이벤트',실행함수());`
    * `btn.addeventListener('click',function(){ alert(); });` //매개변수 있을 땐 이 방식으로 사용
    * `btn.addeventListener('click', func1);` // 외부에 아래(1)처럼 매개변수가 없는 함수가 있을 때 바로 호출 가능
    * `function func1(){}` //(1)
    * 장점) 반복 활용 가능, 내부, 외부 어디든 자유롭게 사용 가능
    * 단점) 위 1번에 비해 쓰기가 조금 복잡함.
